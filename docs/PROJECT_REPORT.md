# 项目报告书

## 项目名称

智研擂台——基于 OpenClaw、飞书与混合算力的多智能体论文创新辩论系统。

## 项目概述

本项目面向科研论文调研、技术路线分析和创新点挖掘场景，构建了一套基于 OpenClaw 的多智能体协作系统。系统以飞书机器人作为自然语言交互入口，以 OpenClaw 作为智能体编排与任务调度平台，结合 StepFun 阶跃星辰模型的长上下文、工具调用和复杂推理能力，实现论文检索、内容阅读、观点生成、交叉质疑、方案比较和创新点总结。

传统论文调研通常依赖人工搜索和逐篇阅读，耗费大量时间。单个大模型虽然能够总结论文，但容易顺着用户的初始想法回答，缺少反方论证，也可能把已有论文中的方法误认为新的创新点。本项目将传统的单模型回答升级为“独立研究—对抗辩论—统一评审”的协作机制。研究员 A 与研究员 B 首先在相互隔离的工作空间中独立搜索资料、分析研究缺口并提出候选方案；之后围绕创新性、最近工作重复度、技术可行性、数据依赖、算力成本和论文价值开展多轮辩论；最后由主智能体担任主持人和评审，融合双方证据并形成结构化论文创新方案。

当前系统包含四个主要 Agent。`main` 负责理解需求、拆分任务、控制流程和输出最终结论；`study` 作为创新研究员 A，负责检索 VLA、具身智能和世界模型论文，从已有工作限制中寻找研究机会；`study-critic` 作为创新研究员 B，独立调研相同范围，并重点审计最近相关工作、实验可行性和潜在失败条件；`health` 展示了系统向饮食记录和健康信息管理等垂直领域扩展的能力。

系统采用分层架构。飞书接入层负责消息输入与结果推送；OpenClaw Gateway 负责身份验证、会话管理和消息路由；多智能体层负责角色隔离、任务分派和辩论协作；技能层提供论文搜索、文档深度阅读、桌面通知和长期记忆；模型层使用 StepFun Step 3.7 Flash 完成长上下文理解和复杂科研推理。本地计算机负责运行网关、Agent工作区、技能脚本、文件处理和记忆服务。

项目采用多项优化方案：两个研究员并行调研以降低等待时间；论文信息经过检索和过滤后再传递给大模型，以减少无关上下文；长论文按章节切分并生成阶段摘要；结构化模板约束报告内容；缓存已读取论文，避免重复请求；为任务设置状态、超时、失败重试和降级机制。系统同时要求智能体区分已验证事实、解释和猜想，所有无法确认的论文信息必须标注为待核验。

## 技术创新点

1. 将对抗辩论引入论文创新点生成，通过反驳暴露方案的隐含假设。
2. 两个研究员先独立工作再交换观点，减少相互复制和群体偏差。
3. 主智能体不承担主要检索，而是专注流程控制、证据检查和方案融合。
4. 使用可插拔 Skills 和独立工作区扩展不同领域能力。
5. 采用本地编排与云端复杂推理结合的混合部署方式。

## NVIDIA 与模型说明

项目当前使用 NVIDIA GeForce RTX 5060 Laptop GPU 所在的本地计算机运行 OpenClaw、文档预处理和工具服务，主推理模型为 `stepfun-plan/step-3.7-flash`。本仓库尚未包含 CUDA、TensorRT-LLM、NVIDIA NIM 或 Nemotron 的实际集成代码。下一阶段计划部署量化后的 NVIDIA 4B 级模型，用于本地摘要、分类、向量化和候选论文重排序，并使用 TensorRT-LLM 降低显存占用。完成集成和性能测试前，NVIDIA SDK部分属于优化规划，不应描述为已经实现。

## 未来展望

后续将完善论文引用自动核验、PDF图表和公式解析、辩论过程可视化、本地NVIDIA模型推理以及DGX Spark适配。系统还可扩展到专利分析、项目评审、技术选型和工业研发决策等场景。
